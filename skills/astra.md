## 🎯 What Do You Need?

Pick your situation:

**I don't know where to start**
→ Use the Path Decision Tree (2 min)
```
Q1: Is the goal clear?
├─ NO → Need broad exploration → EXPLORATORY PATH
└─ YES → Q2: Is approach decided?
   ├─ NO → JUDGMENT PATH
   └─ YES → EXECUTION PATH
```

**I need to frame this task**

```

**I'm handing off to someone**
→ Use Handoff Checklist
```
Clarity: ☐ Goal clear ☐ Known/unknown explicit ☐ Next steps unambiguous
Completeness: ☐ Links provided ☐ Constraints listed ☐ Assumptions surfaced
Refinement: ☐ Working notes separated ☐ Recipient read once ☐ Questions answered

## 📊 Quality Dimensions (Evaluate Your Work)

| Dimension | Ask | Matters For |
|-----------|-----|-------------|
| **Clarity** | Is next step obvious? | Handoffs, specs |
| **Refinement** | Are details intentional? | Reusable output |
| **Responsiveness** | Is causality clear? | Workflows, UX |
| **Composability** | Do pieces fit together? | Patterns, systems |
| **Generativity** | Did we explore broadly? | Discovery, learning |
| **Criticality** | What are we questioning? | Decisions, judgment |

---

## 🔄 Path Switch Signals

**Stuck in Exploratory?**  
Switch to Judgment when: problem definition stabilized, enough information to decide

**Stuck in Judgment?**  
Switch to Execution when: decision is clear, tradeoffs understood

**Stuck in Execution?**  
Switch back to Exploratory when: hidden constraint appears, assumption is wrong

---

## ❓ Uncertain? Ask These Questions

1. **What is actually uncertain?**
   - The goal? → Exploratory path
   - The approach? → Judgment path
   - The implementation? → Execution path

2. **What would change my mind?**
   - Need more info? → Keep exploring
   - Need to choose? → Move to judgment
   - Need to build? → Move to execution

3. **Default:** If genuinely unsure, start with Hybrid (bounded exploration loop)

---

## ✅ Common Mistakes & Fixes

| Mistake | Signal | Fix |
|---------|--------|-----|
| Skip exploration when needed | "I think I know..." (not validated) | Do short exploratory pass |
| Over-explore when clear | Keep investigating after core Q answered | Move to judgment/execution |
| Treat execution like judgment | Building, then discovering wrong direction | Validate goal + approach first |
| Treat judgment like exploration | Endlessly comparing options | Set criteria first, then judge |
| Forget to converge | Astra stays static, knowledge lost | Run convergence after each task |
