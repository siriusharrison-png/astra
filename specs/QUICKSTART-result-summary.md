4. **Handoff Checklist** — Ensure clarity before handoff
5. **Reference Analysis Template** — Analyze and apply references
6. **Decision Template** — Make structured choices

11. **Application Types** — Direct/Adapted/Rejected decision framework
12. **Copy-Paste Prompt** — Bootstrap Astra in new projects (30 sec)
13. **Common Mistakes & Fixes** — Diagnose and fix stuck states

## How to Use It

### From Another Project

```bash
# Load QUICKSTART as primary reference
Load /path/to/astra/QUICKSTART.md

# Use any template directly (no other files needed)
- Task Frame Template (3 min)
- Path Decision Tree (2 min)
- Handoff Checklist (5 min)
```

### Full Astra Loading (when needed)

```bash
Load /path/to/astra/QUICKSTART.md (fast entry point)
Then: /path/to/astra/README.md (full context)
Then: Full foundation (protocols, standards, patterns, workflow)
```

### In Your Workflow

1. **Start:** Copy Task Frame Template, fill it (3 min)
2. **Choose path:** Use Path Decision Tree (2 min)
3. **During work:** Reference Quality Dimensions for your path
4. **Handoff:** Use Handoff Checklist (5 min)
5. **End:** Run Convergence Template (5 min)
6. **Decide reuse:** Use Reusability Check
7. **Update Astra:** Add to CHANGELOG via convergence script

## Reusable Insights

### For Astra

**Primary insight:** Quick-access reference improves adoption more than comprehensive documentation.

Add to memory.md:
> When building documentation systems for reuse across projects, front-load the fastest-path use cases (quick decision trees, copy-paste templates) before deep-reference material. Users will dig deeper only after they've experienced one successful use cycle.

### For Other Projects

**Pattern:** Template + Decision Tree + Checklist structure works for any methodology:
- Use when: providing methodology guidance to teams
- How: extract core patterns, create decision trees, build templates
- Result: 80% of use cases covered in < 5 min per scenario
