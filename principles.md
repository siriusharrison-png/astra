# Principles

## Purpose

Astra exists to reduce repeated explanation and to keep cross-project reasoning consistent.

## Principles

- Prefer reusable structure over one-off notes
- Prefer stable language over temporary wording
- Prefer explicit decision paths over vague guidance
- Prefer the lightest useful process
- Prefer English as the primary working language
- Prefer bounded iteration over uncontrolled expansion
- Prefer task-specific paths over a single forced workflow
- Prefer clear boundaries between stable, evolving, working, and archived material
- Prefer decisions that can be reused in future work
- Prefer writing the rule before writing the example

## Working Rule

If a task is clear, do not force exploration.
If a task is unclear, do not skip exploration.

If a task changes shape, change the path.

## Decision Rule

- Use the exploratory path when uncertainty is high
- Use the judgment path when the question is already framed
- Use the execution path when the output structure is already known
- Use the hybrid path when the task has both known and unknown parts

## Language Rule

English is the default working language.
If another language is needed, treat it as secondary and supporting.
