# Memory

## Purpose

This file stores reusable cross-project knowledge that can evolve over time.

## Good Candidates

- repeated preferences
- validated rules
- recurring decisions
- task retrospectives
- useful phrasing patterns
- proven review habits
- repeated path-selection behavior
- terminology that keeps proving useful
- phrasing that consistently improves clarity
- decision shortcuts that remain reliable across projects

## Update Rule

Only add lessons that are likely to remain useful in future work.

## Memory Entry Shape

When adding a memory item, keep it compact:

- name
- why it matters
- when to use it
- date or context

## Memory Hygiene

- Do not store raw project noise
- Do not store unstable one-off preferences unless they repeat
- Do not store things already covered by the stable layer
