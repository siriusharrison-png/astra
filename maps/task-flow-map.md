# Task Flow Map

## Purpose

A task flow map describes the ordered steps required to complete a task.

It focuses on action, decision points, and completion.

## When to Use

Use a task flow map when:

- the task has a clear start and finish
- the sequence matters
- the user needs to complete a goal efficiently
- you want to identify unnecessary steps

## Core Structure

### 1. Task Goal

- what the user wants to complete
- what counts as success

### 2. Steps

List the steps in order.

### 3. Step Details

For each step, capture:

- action
- input needed
- system response
- decision point
- failure point

### 4. Bottlenecks

Identify steps that add delay, confusion, or avoidable effort.

### 5. Simplification Opportunities

Note where steps can be merged, removed, automated, or clarified.

## Output Shape

A task flow map should usually include:

- task goal
- ordered steps
- step table
- bottlenecks
- simplification opportunities
- next actions
