# State Map

## Purpose

A state map describes the possible states of a system, object, or user-facing element.

It helps clarify transitions and edge cases.

## When to Use

Use a state map when:

- the experience changes based on conditions
- loading, empty, error, success, and disabled states matter
- transitions are important
- state-driven UI or workflows need consistency

## Core Structure

### 1. States

List the relevant states.

### 2. Triggers

Describe what causes each state to appear.

### 3. Transitions

Map how the system moves from one state to another.

### 4. Edge Cases

Capture unusual or failure states.

### 5. Recovery

Describe how the system returns to a healthy state.

## Output Shape

A state map should usually include:

- state list
- triggers
- transitions
- edge cases
- recovery rules
