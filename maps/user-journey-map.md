# User Journey Map

## Purpose

A user journey map describes how a user moves through a task over time.

It helps identify:

- user goals
- touchpoints
- actions
- emotions
- pain points
- opportunities
- moments that matter

## When to Use

Use a user journey map when:

- the product flow is multi-step
- the experience spans more than one screen or channel
- the current journey feels unclear or fragmented
- you need to align design, product, and engineering around the same flow

## Core Structure

### 1. Context

- user segment
- scenario
- task goal
- entry point
- success condition

### 2. Stages

Break the journey into ordered stages.

Examples:

- discover
- evaluate
- start
- complete
- confirm
- return

### 3. Stage Details

For each stage, capture:

- user intent
- user action
- system response
- touchpoints
- friction
- emotion
- opportunity

### 4. Pain Points

Identify the points where the user slows down, hesitates, gets confused, or drops off.

### 5. Opportunities

Record where the experience can be simplified, clarified, shortened, or made more confident.

### 6. Outcome

Define what success looks like after the journey is improved.

## Questions to Ask

- What is the user trying to achieve?
- Where does the journey start?
- What happens next?
- Where does the user hesitate?
- What information is missing?
- What is the system doing at each step?
- What would make the next step easier?
- What would make success more obvious?

## Output Shape

A journey map should usually include:

- user
- scenario
- stages
- stage table
- pain points
- opportunities
- success criteria
- open questions

## Layer Guidance

- Keep the structure stable
- Keep the scenario details in the working layer
- Promote recurring journey patterns into memory only when they repeat
