# Convergence Template

## Purpose

Use this template at the end of a task to capture reusable lessons and decide what should be updated in Astra.

If the lesson is reusable, also record the final convergence result in `CHANGELOG.md`.

## When to Use

Use this template when:

- a task has finished
- there is enough context to review the outcome
- there is useful information worth preserving
- you want to improve future work, not just close the task

## Output

### 1. Task Review

- task:
- path used:
- outcome:
- what worked:
- what did not work:

### 2. Reusable Insights

- insight 1:
- insight 2:
- insight 3:

### 3. Layer Assignment

- stable:
- evolving:
- working:
- archive:
- ignore:

### 4. Update Plan

- file to update:
- change type:
- summary of change:
- confirmation needed:

### 5. Final Convergence Note

Write a short English note with:

- what was learned
- where it belongs
- why it matters for future work

## Writing Rules

- keep it short
- write in English first
- avoid project-specific noise
- prefer reusable language
- prefer clear layer assignment

## Example Shape

```text
task: refine the discussion workflow
path used: hybrid
outcome: clearer task framing and faster convergence
what worked: path selection before execution
what did not work: too much exploratory depth for a clear logic task
insight 1: use the judgment path when the target is already framed
insight 2: keep convergence short and reusable
stable: no change
evolving: add path-selection note
working: archive the task review
archive: save the discarded draft
ignore: raw brainstorming with no reusable value
file to update: /Users/ppio-dn-275/Desktop/astra/memory.md
change type: add evolving note
summary of change: record the path-selection rule
confirmation needed: no
```
