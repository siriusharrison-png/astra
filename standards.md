# Standards

## Scope

This file contains the stable conventions that should rarely change.

## Standards

- Keep content cross-project and reusable
- Keep structural rules separate from working notes
- Keep task-specific material out of the stable layer
- Keep terminology consistent across files and sessions
- Keep updates in English first
- Keep file names and section names short, explicit, and stable
- Keep the top-level vocabulary limited and intentional
- Keep the stable layer readable without extra explanation

## Change Policy

- Change carefully
- Confirm before modifying stable definitions
- Do not use this layer for temporary discussion
- If a change would alter the meaning of the foundation, pause and confirm first
- If a change only improves clarity without changing meaning, it may be added to the evolving layer
